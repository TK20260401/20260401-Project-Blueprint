# MVPを朝から夕方まで磨き続けた日の記録（v0.26.0）

## 朝：TestFlightのフィードバックが溜まっていた

子ども向けマネー教育アプリ「おこづかいクエスト」を開発しています。MVPとしてTestFlight配信中で、少しずつテストユーザーが集まってきました。

朝、集まったフィードバックを眺めながら、ひとつずつ対応していくことに決めました。結果として、**1日で44コミット**を積むことになりました。

この記事は、その1日の記録です。自分用のメモでもあり、似た立場の誰かの参考になれば、という気持ちでまとめています。


## ダークモードで漢字が消えた

最初に片付けたのは、ダークモードの視認性でした。

> 「ダークモードに切り替えたら、漢字が全然読めない」

ライトモードで問題なかったテキストが、ダーク背景だと沈んで見えなくなる、という現象。よくある問題ですが、自分で触ってみたら確かに読めなくて、これは言われるまで気づけなかったなと思いました。

カラーパレットを本文・副次・透かしの3階層に整理し直して、コントラスト比をちゃんと計算してから当て直しました。WCAGのAA基準を下回らないことだけは最低ラインとして守ったつもりです。


## 「どの画面を見てるのか分からない」

次に刺さったフィードバック。

> 「ログインして親モードに入ったあと、いまどの画面にいるのか分からない」

最初は意味が分からなかったけど、実機で触ってみて納得。ヘッダーにタイトルが無くて、視覚的な手掛かりがない。

UXの教科書に載っている「基本のキ」のはずなのに、開発者本人は構造を把握している側なので気づけません。ユーザーにとっては情報が欠けている、という事実。

全画面にヘッダーを統一して、アイコン＋タイトル＋補足の3要素で何の画面かを一目で分かるようにしました。


## ボタンが消えた事件

1日でいちばん頭を抱えたのが、

> 「子ども画面の『ふやす』ボタンが無い。押しても何も起きない」

コードを読むと存在している。なのに画面に出ない、押しても反応しない。

原因はReact NativeのTouchableネスト問題でした。財布カード全体を外側のTouchableOpacityで囲んでいて、内側の「ふやす」ボタンの押下イベントが、親ハンドラに吸い上げられていた。結果、タップすると別画面に飛んでしまう。ユーザーからは「何も起きない」ように見える。

こういうバグはユーザーには絶対説明できない。「押したのに何も起きない」で終わる。実機テストの恐ろしさを改めて思い知った件でした。


## 3pot のリブランドという冒険

午後、思い切ったブランド変更に踏み切りました。

これまで子どもが触れる「使う・貯める・増やす」という3つのポット（仕分け先）を、

- **冒険**（ぼうけん）クエストで稼ぐ
- **取引**（とりひき）商人と取引する
- **金庫**（きんこ）たからをしまう
- **錬成**（れんせい）お金をそだてる

という、よりRPG世界観の強い4つの名詞に統一しました。

子どもには「使う」より「取引」の方が抽象度が高くて難しい。だからこそ、補強サブテキストでサポートします。「取引」の下には「しょうにんと とりひき」、「金庫」の下には「たからを しまう」、というふうに動詞も併記する。

漢字を読めない年齢でも、**世界観で意味が伝わるように**、ふりがなを丁寧に。子どもは新しい漢字を、こうやってRPGの中で覚えていきます。

これは結果的に教育的価値も生んでいて、「お金を扱う」という抽象概念に**冒険・宝庫・錬成**というファンタジーの語彙を被せることで、お金が単なる数字でなく「自分が扱う特別な資源」として体感できるようになる、はず。


## SVGの寸法差問題

ボタンを4つ並べたとき、ラベルの上下位置がどうしても揃わない。Ruby（ふりがな）の実装をいくら直しても直らなかった。

> 「冒険：漢字の位置がずっと低いまま」「錬成：ルビの位置が低い」

途中で気づいたのが、実は**SVGアイコンの寸法差**が犯人だったこと。

```
PixelSwordIcon  : 4×6 grid（縦長）
PixelCartIcon   : 5×5 grid（正方形）
PixelPiggyIcon  : 6×6 grid（正方形）
PixelChartIcon  : 5×6 grid（縦長）
```

`size={24}` で呼んでも、内部gridの縦横比が違うため実描画寸法がバラバラ。直下のテキストがそこから始まるので、当然テキスト位置もバラバラに。

対策は地味で、各アイコンを `width: 24, height: 24` の固定枠（IconBox）でラップしただけ。それで全部揃いました。

> 「やっとそろったよ」

このひとことが今日いちばん嬉しかったかも。


## 戻るボタンをHCD準拠で整理

戻るボタンが画面ごとにバラバラだった問題。

- 🏠 アイコンなのに「前の画面に戻る」動作 → 認知モデルとズレる
- ある画面では1段戻り、ある画面ではTOP直行、ある画面ではモーダル閉じ
- 補強テキストもなく、押してから「あれ、ここどこ？」となる

**HCD（人間中心設計）的に整理**して、4階層のルールにしました。

| アイコン | 動作 | 補強 |
|---|---|---|
| 🏠 | TOPに戻る | (TOPへ) |
| 🚪 | 1段戻る | (まえへ) |
| ✕ | モーダル閉じる | とじる |
| ← | wizard step戻り | （矢印で自明） |

子ども向けアプリだからこそ、補強で「どこに行くのか」を絶対に明示する。子どもの「あれ、押すと何が起きるの？」という不安を取り除くために。


## ペットを愛でる演出

夕方は、ペットシステムにインタラクションを足しました。

ペットをタップすると、ハートが3つ飛んで、ペットがちょっと跳ねる。それだけの演出ですが、「ペットが喜んでくれた」感が出ます。

ペットが進化（egg→baby→child→adult）した瞬間には、ゴールドのフラッシュとスケールアップのアニメーション。これは滅多に起きないイベントだからこそ、ちゃんと祝う。

親が承認した瞬間にも、画面中央に大きな✓と「しょうにんしました！」のバナーが出るようにしました。承認って親にとっては毎日の作業で、何度もやるからこそ、ちゃんと「やった！」という気持ちになれる小さなフィードバックを足したかった。


## 1日を振り返って

44コミットを積んだと書きましたが、一つひとつは小さい修正です。大きな新機能を実装した日ではなく、**MVPを「ちゃんと使える状態」に磨いた1日**でした。

MVPは機能追加よりも、いま動いている部分を壊さず、ちゃんと見える・使える・気持ちよくするほうが、検証の質が上がると感じています。ユーザーが「使いにくい」と感じる原因の大半は、今日直したような小さいディテールにある。

ベンチャーに転職してまだ日が浅く、教育コンテンツ制作は初めての領域ですが、こうやって地道に直しては検証することを繰り返していけるのが、いまのフェーズでは何より大事だと思っています。


## ここからはユーザーの声を聞く段階

44コミットを積み上げて、今日時点で**外部に公開しても大丈夫なレベル**にはなったと感じています。基本的な使い勝手は整い、大きな破綻もない。3potのリブランドも、戻るボタンの導線も、ペットの演出も、ひととおり動く状態にたどり着きました。

ここから先は、自分が「こうだろう」と想像で手を動かすフェーズではないと思っています。

MVPを触った親子が「どこで止まったか」「何が分からなかったか」「逆にどこが面白かったか」。そうしたリアルな反応が、これ以上ないくらい価値のあるヒントになるフェーズに入りました。自分の頭の中だけで考え続けても、もう次の一手が出てこない感覚があります。
