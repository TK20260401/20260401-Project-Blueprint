# 時計アプリで学ぶ アルゴリズム問題集

NHK風時計アプリの実装を題材に、共通テスト「情報I」のアルゴリズム分野を学ぶ問題集である。
観点別（知識・理解 / 思考・判断・表現 / 主体的に学習に取り組む態度）に構成している。

---

## 知識・理解

---

### A-K1（難易度1）

時計の秒針は60秒で1周（360度）する。秒数sから角度を求める式として正しいものはどれか。

| 選択肢 | 内容 |
| --- | --- |
| A | s × 6 |
| B | s × 60 |
| C | s / 6 |
| D | s × 360 |

#### 正解: A

> 360度 ÷ 60秒 = 6度/秒。したがってs秒のときの角度はs × 6度である。s=15なら90度（3時の位置）となる。

---

### A-K2（難易度1）

時計の目盛りは全部で60本あり、5本に1本が長い目盛り（時刻マーカー）である。長い目盛りの本数はいくつか。

| 選択肢 | 内容 |
| --- | --- |
| A | 12本 |
| B | 5本 |
| C | 10本 |
| D | 60本 |

#### 正解: A

> 0, 5, 10, 15, …, 55の12個が5の倍数である。これは12時間分の時刻位置に対応する。判定式は`i % 5 === 0`で表現できる。

---

### A-K3（難易度1）

度数法の角度をラジアンに変換する式として正しいものはどれか。

| 選択肢 | 内容 |
| --- | --- |
| A | ラジアン = 度 × π / 180 |
| B | ラジアン = 度 × 180 / π |
| C | ラジアン = 度 × π |
| D | ラジアン = 度 / π |

#### 正解: A

> 360度 = 2π ラジアンなので、1度 = π/180 ラジアン。選択肢Bはラジアンから度への逆変換である。

---

### A-K4（難易度1）

円の中心(CX, CY)から角度θ（ラジアン）、距離Rの位置の座標(x, y)を求める式として正しいものはどれか。

| 選択肢 | 内容 |
| --- | --- |
| A | x = CX + R × cos(θ)、y = CY + R × sin(θ) |
| B | x = CX + R × sin(θ)、y = CY + R × cos(θ) |
| C | x = CX × cos(θ)、y = CY × sin(θ) |
| D | x = R × cos(θ)、y = R × sin(θ) |

#### 正解: A

> 三角関数の定義より、半径Rの円上の点はcos（横方向）とsin（縦方向）で求まる。選択肢Dは中心が原点の場合のみ正しい。

---

### A-K5（難易度2）

Canvas APIで角度0ラジアンは右方向（3時位置）を指す。時計の12時方向を起点にするための補正として正しいものはどれか。

| 選択肢 | 内容 |
| --- | --- |
| A | 角度から90度を引く |
| B | 角度に90度を足す |
| C | 角度に180度を足す |
| D | 角度を2倍にする |

#### 正解: A

> 12時は3時から反時計回りに90度の位置にある。角度から90度を引くことで補正する。コード上では`(s * 6 - 90)`と記述している。

---

### A-K6（難易度2）

以下の手順でミリ秒msを「時:分:秒」に変換する。各手順の計算で使う演算として正しい組合せはどれか。

```
手順1: totalSec ← ms ÷ 1000 の整数部分
手順2: h ← totalSec ÷ 3600 の整数部分
手順3: m ← (totalSec % 3600) ÷ 60 の整数部分
手順4: s ← totalSec % 60
```

| 選択肢 | 内容 |
| --- | --- |
| A | 手順2は整数除算、手順4は剰余演算 |
| B | 手順2は剰余演算、手順4は整数除算 |
| C | すべて整数除算 |
| D | すべて剰余演算 |

#### 正解: A

> 時を求めるには秒数を3600で割る（整数除算）。秒を求めるには60で割った余り（剰余演算）を使う。整数除算と剰余演算の使い分けが時間変換の要である。

## 思考・判断・表現

### A-T1（難易度2）

時針は1時間で30度、1分で0.5度進む。現在3時20分のとき、時針の角度（12時を0度とする）として正しいものはどれか。

| 選択肢 | 内容 |
| --- | --- |
| A | 100度 |
| B | 90度 |
| C | 110度 |
| D | 120度 |

#### 正解: A

> 3時 × 30度 = 90度、20分 × 0.5度 = 10度。合計100度。90度は3時ちょうどの値で分の進みを未考慮。

---

### A-T2（難易度2）

中心(200, 200)、半径180の円上で角度90度（真下、6時位置）の目盛りの外側端座標を求めよ。ただし角度は「12時=0度、時計回り」で、Canvas補正済み（-90度補正後）とする。補正後の角度は0度（ラジアン0）である。cos(0)=1、sin(0)=0とする。

| 選択肢 | 内容 |
| --- | --- |
| A | (380, 200) |
| B | (200, 380) |
| C | (200, 20) |
| D | (20, 200) |

#### 正解: A

> x = 200 + 180 × cos(0) = 200 + 180 = 380。y = 200 + 180 × sin(0) = 200。これは3時位置（右端）の座標である。

### A-T3（難易度3）

ms = 7325000のとき、時分秒変換の手順で求まる値の組合せとして正しいものはどれか。

```
手順1: totalSec ← 7325000 ÷ 1000 → 7325
手順2: h ← 7325 ÷ 3600 → ?
手順3: m ← (7325 % 3600) ÷ 60 → ?
手順4: s ← 7325 % 60 → ?
```

| 選択肢 | 内容 |
| --- | --- |
| A | h=2, m=2, s=5 |
| B | h=2, m=0, s=25 |
| C | h=2, m=2, s=25 |
| D | h=1, m=62, s=5 |

#### 正解: A

> h = 7325 ÷ 3600 = 2（余り125）。m = 125 ÷ 60 = 2（余り5）。s = 5。選択肢Cのs=25は手順3の余りと手順4を混同している。

### A-T4（難易度3）

カウントダウンタイマーで全体T=180000ms、経過E=135000msのとき、残り率pctはいくつか。

```
remain ← max(0, T - E)
pct ← remain ÷ T × 100
```

| 選択肢 | 内容 |
| --- | --- |
| A | 25 |
| B | 75 |
| C | 45 |
| D | 15 |

#### 正解: A

> remain = 45000。pct = 45000 ÷ 180000 × 100 = 25。75は経過率であり残り率ではない。

### A-T5（難易度3）

天気コードの条件分岐で、code=50のとき返される値はどれか。

```
もし code = 0 ならば "快晴"
そうでなくもし code ≦ 3 ならば "晴れ"
そうでなくもし code ≦ 49 ならば "くもり"
そうでなくもし code ≦ 59 ならば "霧雨"
そうでなくもし code ≦ 69 ならば "雨"
```

| 選択肢 | 内容 |
| --- | --- |
| A | "霧雨" |
| B | "くもり" |
| C | "雨" |
| D | 条件に合致しない |

#### 正解: A

> code=50は49以下でないが59以下である。上から順に評価され、最初に真となった`code ≦ 59`で"霧雨"が返される。

### A-T6（難易度3）

6回のビープ音を0.5秒間隔で鳴らす。各ビープは0.3秒間。開始時刻t(i)=i×0.5（i=0,...,5）のとき、最初の開始から最後の終了までの総時間はどれか。

| 選択肢 | 内容 |
| --- | --- |
| A | 2.8秒 |
| B | 3.0秒 |
| C | 3.3秒 |
| D | 1.8秒 |

#### 正解: A

> 最後のビープはt(5)=2.5秒に開始し0.3秒間。2.5+0.3=2.8秒。3.0は6×0.5でビープの長さを未考慮。

### A-T7（難易度3）

タイマーの経過時間計測で、10msごとに固定値+10する方法と、Date.now()の差分を使う方法がある。後者が優れている理由として正しいものはどれか。

| 選択肢 | 内容 |
| --- | --- |
| A | setIntervalの実行間隔は正確でないため、実際の経過時間との累積誤差を防げる |
| B | Date.now()の方が計算が速い |
| C | +10する方法ではオーバーフローが発生する |
| D | Date.now()は小数点以下も返すため精度が高い |

#### 正解: A

> setIntervalはOSスケジューリングやタブ非アクティブ化により正確に実行されない。固定値加算では累積誤差が大きくなるが、Date.now()の差分なら実際の経過時間を正確に反映する。
