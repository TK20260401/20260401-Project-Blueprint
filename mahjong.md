## プロジェクト構想案：自律学習支援Webアプリ

| 項目 | 内容 |
| :--- | :--- |
| **タイトル** | **LogicRiichi（ロジリーチ）ー 思考の可視化と自律学習プラットフォーム** |
| **目的** | 麻雀の戦略的思考（確率・統計）を入り口に、「情報Ⅰ」の知識や「応用情報（AP）」のアルゴリズムを、AIの伴走により自走して習得する。 |
| **主要な機能一覧** | 1. **麻雀牌によるアルゴリズム学習**: 牌の並べ替え（ソート）や待ち牌検索（探索）を、プログラムコード（Python）で実装・シミュレーションする。<br>2. **AI対局アノテーション**: 自分の打牌ログを保存し、AIが「なぜその一手か」を論理的（LOGICAL）に問いかけ、思考を言語化させる。<br>3. **資格試験クイズ・ダッシュボード**: 情報ⅠやAPの過去問を解くと、麻雀の役やレベルに変換されるゲーミフィケーション要素。<br>4. **アクセシビリティ・レイヤー**: 肢体不自由や高次脳機能障害があっても、視線や最小限のスイッチで操作可能なUI設計。 |
| **技術スタック** | **Frontend**: Next.js (TypeScript) <br>**Backend/DB**: Supabase (PostgreSQL, Auth, Realtime) <br>**AI**: Claude API / OpenAI API (思考分析・ヒント生成) <br>**Infrastructure**: Vercel |

-----

## 1つ目に作る「アジャイル・プロトタイプ」の定義

まずは、**「麻雀の待ち牌計算アルゴリズムを、Python/TypeScriptで解く学習クイズアプリ」** を作成

### なぜこれか？

  * **情報Ⅰ・資格対策に直結**: 「多次元配列」「ループ処理」「条件分岐」の基礎がすべて詰まっている。
  * **自走自律の検証**: 生徒がコードを書く際、AI（Claude）が「答え」ではなく「デバッグのヒント」を出す仕組みをSupabase経由で実装できる。
  * **拡張性**: これができれば、後に「VR麻雀」や「AI対局」へスムーズにスケールできる。
