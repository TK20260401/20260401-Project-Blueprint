# Logic-Riichi マーケティング・バズ施策

> 目的: 「麻雀 × アルゴリズム」という稀有なポジションを使い、SNS・技術記事で着火させ、流入を定着に変える。
> 関連: 機能要件は `docs/requirements.md`（OGP・公開API・デイリー問題などのバズ受け皿）。
> 作成: 2026-06-19

---

## 1. 市場の追い風（なぜ「今」か）

- **Mリーグ** 配信は平均約100万視聴。麻雀は「賭博」から「頭脳スポーツ」へ再定義され、若年層・家族連れにまで拡大。
- **雀魂/じゃんたま** が収益シェア50%超を維持。若年層・女性に強く、段位アカウントは160万超。
- **麻雀AI**（Suphx → NAGA → 2025年雀魂統合のMAKA）は常時ホットトピック。「AIは何を切る/何待ちか」は永続的関心事。
- アルゴリズム記事（シャンテン数・待牌探索）は Qiita/Zenn で需要が枯れない定番。tomohxx・kobalab・KamichanR らの高速計算記事が競い合う土壌が既にある。

## 2. ターゲットクラスタ（拡散の核・着火順）

| 優先 | クラスタ | どこにいるか | 刺さる切り口 |
|---|---|---|---|
| **A（着火点）** | 麻雀を打つエンジニア | X / Qiita / Zenn | 「自分も書きたかった待ち判定」を高完成度で提示 |
| **A** | 競プロer・アルゴリズム好き | X / AtCoder界隈 | O(1)・ビット演算の最適化バトル |
| **B** | 情報系学生・初学者 | X / Qiita / note | 身近な題材で学ぶ段階式実装＝学習/就活ネタ |
| **B** | 見る雀で論理に興味がある層 | X / YouTube | 「何切る/何待ち」をアルゴリズムで説明 |
| **C** | 麻雀インフルエンサー/Vtuber | X / YouTube / Twitch | 配信で遊べるクイズ |
| **C** | 教育・EdTech関係者 | X / note / はてブ | 「選択式→虫食い→自由記述」の段階式設計論 |
| **D** | 一般雀魂層 | TikTok / X | スクショから待ちを当てる即時性・ゲーム性 |

> 最重要インサイト: A「麻雀 × エンジニア」は人口少だが**発信力・引用力が桁違い**。最初の1本はA層に全振りし、はてブ→Xトレンドの導火線にする。

## 4. 投下プラン（4週間）

| 週 | 施策 | 狙い |
|---|---|---|
| **Week 0（仕込み・最重要）** | 動的OGP＋牌姿シェア画像を実装／公開API（待ち判定・レート制限）＋ドキュメント整備／クイズ5問ストック・Xアカウント整備 | 全SNS施策の前提。これが無いと効果半減 |
| **Week 1（着火）** | 企画1をZenn投稿＋Qiita要約版（**月曜朝7-8時・タイトル25字前後・本文1万字**）。末尾に「解けるアプリ」「叩けるAPI」導線。既存著者記事を引用リスペクト | はてブ100超＝着火成功。Xトレンド波及 |
| **Week 2（持続）** | 企画7ショート3本＋企画4クイズを毎朝1問。反応を見て企画11の続編準備 | デイリーで母数を作る |
| **Week 3（拡大）** | 企画6（雀魂スクショ→待ち判定）動画をX/TikTok。企画3でAI話題に接続。インフルエンサー/Vtuberに配信用クイズを直接案内 | 麻雀クラスタ攻略 |
| **Week 4（資産化）** | 企画12（全漢字ルビ・段階式）をnote/はてブ。流入をデイリー問題・リーダーボードで定着 | EdTech層＋SEO資産 |

**KPIの考え方**: Week1で「はてブ100超＝着火成功」。失敗しても Week2 のデイリー＋ショートで母数を作り、Week3 で麻雀クラスタへ。1本に賭けず、瞬発（記事）× 持続（デイリー/ショート）× 拡大（麻雀クラスタ）の三層で回す。
