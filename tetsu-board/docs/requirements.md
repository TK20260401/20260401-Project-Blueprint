# tetsu-board 要件定義書（次期版 / To-be）

> 鉄道テーマの教育ボードゲーム。第1弾は知的障害・発達系を含む特支／夜間定時制（C層）向け。教科横断（地理・経済・公民・情報・家庭・数学）の学習効果をボードゲームの楽しさで担保する。
> 位置づけ: 試作版（2026-05〜実装中）からの **To-be（改善反映の次期要件）**。法務調査・競合分析（`legal-analysis.md` ＋ ルート `PRODUCT-STRATEGY-REPORT.md`）を反映。
> リリース方針: **2027年（来年）リリース目標**。長い助走を活かし、差別化・法務クリアランス・コンテンツ拡充を段階的に積む。当面は外部公開・マネタイズせず形を整える方針を継続。
> 最終更新: 2026-06-19

---

## 0. プロダクト方針（要件の前提）

### ポジショニング（事業リスクを踏まえた差別化）
- コナミ「桃鉄 教育版」が**全国小学校の約35%（6,800校超）に無償導入済み**。汎用的な地理学習ゲームでは正面衝突で勝ち目が薄い。
- tetsu-board の活路＝本家が手薄な **「特別支援・夜間定時制（C層）・教科横断」**。「無償の桃鉄でなく、なぜ tetsu-board か」を最初から差別化軸として設計に埋め込む。

### 設計の不変原則（法務・手戻り防止）
1. **完全独自名称・独自ビジュアル**（先行作品名をマーケ・テキスト・コード・ドキュメントから排除）。
2. **ゲームロジックは副作用なしの純関数**（`lib/game/engine.ts`）に分離。永続化・認証・Realtime は後付けで store のアクションを Server Action に差し替えるだけにする。
3. **お邪魔キャラは最も独自化リソースを投入する要素**（名称・容姿・色・性格・演出を本家から明確に離す）。

## 2. 非機能要件（Non-Functional Requirements）

### 2-1. 法務・コンプライアンス（リリース前に必須クリア）
- **NFR-T-L01 著作権**: メカニクスはアイデア層で保護対象外（釣りゲータウン2・放置少女事件で補強）。ただし**具体的表現（マップ/キャラ/UIのビジュアル）は独自**を保つ。
- **NFR-T-L02 お邪魔キャラ**: 著作権・不競法の唯一の争点。本家ボンビーから名称・容姿・色・性格・演出を明確に離す（第三者レビュー）。
- **NFR-T-L03 商標**: 正式名称を J-PlatPat で先行調査（第9/28/41類）、本家想起ワード（電鉄/鉄道すごろく等）回避、外部公開前に出願検討。
- **NFR-T-L04 鉄道事業者**: 社章/ロゴ・路線カラー・駅メロディ・公式ご当地キャラ・車両意匠は不使用。駅名・地名はテキストの地理表示に限定。
- **NFR-T-L05 地図データ**: 出所を確定し出典表記を実装（地理院＝出典＋加工明記／OSM＝© OpenStreetMap contributors＋ODbL継承の検討）。
- **NFR-T-L06 教育機関展開**: 著作権法35条は学校の授業利用を守るだけで自社プロダクトを正当化しない。**自前で権利クリアランスを完結**。素材（フォント/BGM/効果音/アイコン）のライセンス台帳を整備。
- **NFR-T-L07 体制**: 最初の対外公開／有償納品の直前に**法人化**（責任限定・契約主体・知財帰属）。自治体/教委納品の知財非侵害保証に署名できる証跡を整備。

### 2-2. アクセシビリティ／UD（C層対応の中核）
- **NFR-T-U01**: 全漢字ルビ、UDフォント、最小タッチターゲット、色＋形の区別、`prefers-reduced-motion`。
- **NFR-T-U02**: 失敗を罰しない（減点はあってもゲームオーバー/否定演出を避け、再挑戦を促す）。
- **NFR-T-U03**: 手順の分解・視覚的提示、制限時間の可変・無制限モード。

### 2-3. 保守性・拡張性（手戻り防止）
- **NFR-T-O01**: ゲームロジックは純関数（`lib/game/*`）に分離し vitest で回帰テスト（現状57件）を維持・拡充。
- **NFR-T-O02**: 永続化・認証・Realtime は store アクションの差し替えのみで後付け可能な構造を保つ。
- **NFR-T-O03**: クイズ・駅・カードはデータ駆動（プール/デッキ）で増減可能。

### 2-4. 性能
- **NFR-T-P01**: マップ生成・BFS（最短距離/誘導線）は決定的かつ即応。縦長盤面のスクロール描画が滑らか。

## 3. 技術要件（Technical Requirements）

### 3-1. システム構成
- フロント＝Next.js（App Router）/ React / TypeScript / Tailwind CSS v4 / Zustand。
- ゲームエンジン＝純関数（`lib/game/engine.ts` rollDice/walk/continueFromBranch/judgeAnswer/calcScore/shortestPath/pickDestination 等）。
- マップ＝`generateMap.ts` / `rng.ts` / `stationPool.ts` / `japanMap.ts`（緯度経度配置・外周ループ・フェリー航路）。
- カード＝`cards.ts`（drawCard/applyCard）。
- 状態管理＝`store/gameStore.ts`（Zustand、状態遷移は engine に委譲）。
- テスト＝vitest（純関数・マップ生成の回帰）。

### 3-2. 次期追加（リリースに向け）
- **TR-T01**: Supabase 接続（認証・RLS・進捗/スコア永続化）。Server Action 経由でエンジンをサーバ移植。
- **TR-T02**: 地図データ源の確定とクレジット表記欄の実装。
- **TR-T03**: 教科横断クイズの分野タグ・到達度集計のデータモデル。
- **TR-T04**: （将来）Realtime マルチプレイ or 学級内利用モード。

### 3-3. 外部連携
- **TR-T-E01**: Supabase（認証・DB・Realtime）。
- **TR-T-E02**: 地図データ（地理院 or OSM）— ライセンス順守と出典表記。

## 4. スコープ外 / 将来
- 外部公開・マネタイズは2027リリースまで段階的（当面はゲームの形を整える）。
- 物理ボード版・オンライン対戦は P2。

## 5. 2027リリースに向けたマイルストーン（要件ベース）
1. コアループの磨き込み（FR-T10/T11/T13 の教育・C層差別化を要件水準まで）。
2. **お邪魔キャラの独自デザイン確立**（FR-T23 / NFR-T-L02）。
3. 法務クリアランス（NFR-T-L01〜L07 のチェックリスト消化、J-PlatPat・地図出典・素材台帳）。
4. 永続化・認証の後付け（FR-T30）。
5. 法人化と差別化ストーリーの営業資料化（NFR-T-L07）。
