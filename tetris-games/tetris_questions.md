### 問1（難易度1）繰り返し構造の終了値

**分野:** コンピュータ・プログラミング

テトリスではブロックが1マスずつ下に落下する。次の疑似言語コードはブロックのy座標を1ずつ増やしながら落下させる処理である。

```
y ← 0
y < 20 の間繰り返す
　　y ← y + 1
繰り返し終わり
```

この繰り返し構造が終了するときのyの値はいくつか。

| 選択肢 | 内容 |
| --- | --- |
| A | 19 |
| B | 20 |
| C | 21 |
| D | 0 |

**正解: B**

> yは0から始まり毎回1加算される。y<20が偽になるのはy=20のときなので、終了時のyは20である。y=19のときはまだ条件が真なので繰り返しが続く。

### 問2（難易度2）落下条件の空欄補充

**分野:** コンピュータ・プログラミング

テトリスのブロック落下処理で、盤面の底または他ブロックに接触するまで落下を続けたい。次のコードの空欄を埋めよ。

```
y ← 0
（　ア　）の間繰り返す
　　y ← y + 1
繰り返し終わり
```

盤面の高さは20行（添字0〜19）、board[y+1]が0なら空きマスとする。

| 選択肢 | 内容 |
| --- | --- |
| A | y < 19 かつ board[y+1] = 0 |
| B | y < 20 かつ board[y] = 0 |
| C | y <= 20 または board[y+1] = 0 |
| D | y < 19 または board[y+1] = 0 |

**正解: A**

> 落下は盤面の底（y=19）の手前まで、かつ次のマスが空きの間だけ続ける必要がある。y<19かつboard[y+1]=0が正しい条件である。「または」では片方が真なら止まらないため不適切。

### 問3（難易度3）ハードドロップの計算量

**分野:** コンピュータ・プログラミング

テトリスのハードドロップ（一気に落下）を高速化したい。盤面の高さをH、幅をWとする。単純にy=0から1マスずつ衝突判定しながら落下させると計算量はO(H)である。各列について「その列の最も高いブロックの位置」を配列heightsに保持しておくと、ハードドロップの計算量はどうなるか。

| 選択肢 | 内容 |
| --- | --- |
| A | O(H×W) |
| B | O(W) |
| C | O(1) |
| D | O(H+W) |

**正解: B**

> テトリミノの各ブロックが占める列のheights値を参照し、最小の空き位置を求める処理はテトリミノの幅（最大W）に比例する。heights配列を事前計算しておけば1マスずつ走査するO(H)よりO(W)で済む。O(1)にはならない。

### 問4（難易度1）配列の90度回転

**分野:** コンピュータ・プログラミング

テトリスではブロックを90度回転させる。2×2の配列Aに格納されたブロック形状を時計回りに90度回転させたい。元の配列がA[0][0]=1, A[0][1]=0, A[1][0]=1, A[1][1]=1のとき、回転後のA[0][0]の値はいくつか。

| 選択肢 | 内容 |
| --- | --- |
| A | 1 |
| B | 0 |
| C | 1と0の両方あり得る |
| D | データ不足で不明 |

**正解: A**

> 2×2配列の時計回り90度回転では、回転後の[i][j]は元の[N-1-j][i]となる。回転後A[0][0]=元のA[1][0]=1である。0とする誤りは変換式の行列の向きを間違えた場合に起こる。

### 問5（難易度2）回転の変換式

**分野:** コンピュータ・プログラミング

テトリスのブロック回転処理で、N×Nの配列を時計回りに90度回転させる疑似言語コードがある。

```
i を 0 から N-1 まで 1 ずつ増やしながら繰り返す
　j を 0 から N-1 まで 1 ずつ増やしながら繰り返す
　　B[i][j] ← A[（　ア　）][（　イ　）]
　繰り返し終わり
繰り返し終わり
```

空欄（ア）（イ）に入る組合せはどれか。

| 選択肢 | 内容 |
| --- | --- |
| A | ア: N-1-j　イ: i |
| B | ア: j　イ: N-1-i |
| C | ア: i　イ: N-1-j |
| D | ア: N-1-i　イ: j |

**正解: A**

> 時計回り90度回転ではB[i][j]=A[N-1-j][i]が正しい変換式である。反時計回りの場合はB[i][j]=A[j][N-1-i]となり、選択肢Bはそちらに該当する。

### 問6（難易度3）壁蹴りの計算量

**分野:** コンピュータ・プログラミング

テトリスのSRS（スーパーローテーションシステム）では、回転が壁や他ブロックに衝突した場合、複数のオフセット位置を順に試す「壁蹴り」を行う。5つのオフセットを順に試す場合、1つのオフセットの衝突判定にO(K)（Kはブロックのマス数）かかるとする。最悪計算量はどれか。

| 選択肢 | 内容 |
| --- | --- |
| A | O(K) |
| B | O(5K) |
| C | O(5+K) |
| D | O(K²) |

**正解: B**

> 各オフセットでK個のマスの衝突判定を行い、最悪5回試すので5×K=O(5K)=O(K)となる。ただし定数を明示するならO(5K)が最も正確な記述である。O(K²)はブロック同士の全組合せ比較の場合であり、ここでは該当しない。

### 問7（難易度1）ライン消去の判定

**分野:** コンピュータ・プログラミング

テトリスで横一列がすべて埋まったらその行を消去する。盤面の1行（幅W=10）が埋まっているか判定する次のコードの空欄を埋めよ。

```
filled ← 真
x を 0 から 9 まで 1 ずつ増やしながら繰り返す
　もし board[y][x] = 0 ならば
　　filled ← （　ア　）
　を実行する
繰り返し終わり
```

| 選択肢 | 内容 |
| --- | --- |
| A | ア: 偽 |
| B | ア: 真 |
| C | ア: 0 |
| D | ア: board[y][x] |

**正解: A**

> 1マスでも空き（0）があれば「埋まっていない」ので、filledを偽にする。真を代入すると空きがあっても埋まった判定になってしまう。0を代入するのは論理値でなく整数を使っており不適切。

テトリスのライン消去処理で、盤面（高さH=20、幅W=10）の全行を走査し、埋まった行を消去して上の行を下にずらす疑似言語コードがある。

y を H-1 から 0 まで 1 ずつ減らしながら繰り返す

### 問9（難易度3）複数行消去の計算量比較

**分野:** コンピュータ・プログラミング

テトリスで同時に複数行を消去する場合、消去後に上の行を詰める処理の計算量を考える。盤面の高さH、幅Wのとき、最大4行同時消去でナイーブに毎回全行をシフトする方法の計算量と、消去行をまとめて一度にシフトする方法の計算量の組合せとして正しいものはどれか。

| 選択肢 | 内容 |
| --- | --- |
| A | ナイーブ: O(H²W)、まとめ: O(HW) |
| B | ナイーブ: O(HW)、まとめ: O(HW) |
| C | ナイーブ: O(H²W)、まとめ: O(H²W) |
| D | ナイーブ: O(HW)、まとめ: O(W) |

**正解: A**

> ナイーブに1行消去ごとに全行シフトするとO(HW)が最大4回だがHに比例した行移動があるためO(H²W)。まとめてシフトすれば各行を一度だけ移動するのでO(HW)で済む。

### 問11（難易度2）衝突判定の空欄補充

テトリスの衝突判定で、ブロック（4マス分の座標を持つ）が盤面内の他ブロックと衝突するかを判定する。次のコードの空欄を埋めよ。

```
collision ← 偽
k を 0 から 3 まで 1 ずつ増やしながら繰り返す
　bx ← block[k].x + dx
　by ← block[k].y + dy

### 問13（難易度1）スコアの累積計算

**分野:** コンピュータ・プログラミング

### 問15（難易度3）コンボボーナスの合計

combo ← 0

### 問17（難易度2）7-bagシャッフルの空欄補充

**分野:** コンピュータ・プログラミング

テトリスのブロック生成で、7種類を1巡ずつ偏りなく出す「7-bag」方式を実現したい。次のコードの空欄を埋めよ。

```
bag ← {1,2,3,4,5,6,7}
i を 6 から 1 まで 1 ずつ減らしながら繰り返す

### 問18（難易度3）7-bagと純粋乱数の確率比較

**分野:** コンピュータ・プログラミング

| --- | --- |

### 問19（難易度1）ゲームループの順序

### 問21（難易度3）フレームあたりの計算量

**分野:** コンピュータ・プログラミング

テトリスのゲームループにおいて、1フレームあたりの処理が「入力処理O(1)→衝突判定O(K)→ライン消去判定O(HW)→描画O(HW)」で構成される場合、1フレームあたりの全体計算量として最も適切なものはどれか。ただしK<HWとする。

| 選択肢 | 内容 |
| --- | --- |
| A | O(HW) |
| B | O(K+HW) |
| C | O(H²W²) |
| D | O(K×HW) |

**正解: A**

> 各処理の計算量の合計はO(1)+O(K)+O(HW)+O(HW)=O(HW)である。K<HWなのでO(K)はO(HW)に吸収される。加算の場合は最大のオーダーが支配的になる。O(K×HW)は乗算であり各処理が独立なので不適切。
