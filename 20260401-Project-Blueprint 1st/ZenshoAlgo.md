**「全商情報処理検定 プログラミング部門（1級・2級）」** のアルゴリズム対策に完全にフォーカスしたWebアプリの構成案を作成しました。

この検定の肝である「疑似言語（流れ図・トレース）」を、自らコードを書いて動かしながら理解できる **「デバッグ型学習環境」** を核に据えます。

---

## プロジェクト構成案：Zensho-Algo (ゼンショ・アルゴ)

### 1. タイトル
**Zensho-Algo ー 全商プログラミング検定 完全攻略アルゴリズム・トレーナー**

### 2. 目的
全商情報処理検定の「流れ図（フローチャート）」と「疑似言語」の壁を突破するため、実際にコードを動かして変数の中身を可視化（トレース）し、合格に必要な「論理的思考力」を自律的に養う。

### 3. 主要な機能一覧

| 機能名 | 詳細・役割 | 全商検定への直結要素 |
| :--- | :--- | :--- |
| **全商・疑似言語エディタ** | 検定特有の表記（←, ＝等）でコードを書くと、即座に実行結果を表示。 | 流れ図のアルゴリズム実装力。 |
| **リアルタイム・トレース表** | ループ（反復）ごとに、全変数の値を表形式で自動更新。 | 検定第4問・第5問の「トレース問題」対策。 |
| **スモールステップ・課題群** | 合計、最大・最小、並べ替え（ソート）、探索、集計など頻出パターンを網羅。 | 1級・2級のアルゴリズム実技・筆記対策。 |
| **AIデバッグ・アシスト** | 「ここでiが+1されていない理由は？」等、検定の引っかけポイントをAIが指摘。 | 凡ミスを防ぐ「自律的チェック力」の育成。 |
| **コード⇔フローチャート変換** | 書いたコードを自動で流れ図に変換し、論理構造を視覚化。 | 流れ図の穴埋め問題に対する構造的理解。 |

---

### 4. アルゴリズム特化の学習コンテンツ案（全商1級・2級対応）

16年間の教育現場での知見を活かし、生徒がつまずきやすい「配列操作」を視覚化します。

* **基本アルゴリズムの可視化**:
    * **合計・平均・カウント**: 配列の要素を1つずつ取り出し、変数 `SUM` や `CNT` が積み上がる様子をアニメーション化。
    * **最大・最小値**: 暫定の最大値 `MAX` と現在の要素を比較し、入れ替わる瞬間を強調。
* **ソート（整列）の徹底理解**:
    * **バブルソート・選択ソート**: 隣り合う要素の比較と「交換（ワーク変数への退避）」のプロセスを1ステップずつ実行。
* **多次元配列とコントロールブレイク**:
    * **1級頻出項目**: 地区別・商品別の集計など、キーが変わったタイミングで合計を出力する「コントロールブレイク」の論理構造を視覚化。

### 5. 技術スタック（ベータ版：Vercel / Supabase）

* **Frontend**: **Next.js (Vercel)**
    * UIライブラリ：Tailwind CSS / Lucide-react（アイコン）
    * エディタ：Monaco Editor（全商疑似言語用にカスタマイズ）
* **Backend/Database**: **Supabase**
    * `problems` テーブル：全商の過去問をベースにしたアルゴリズム課題の保存。
    * `user_logs` テーブル：生徒が書いたコードと、エラーを出した箇所の蓄積（分析用）。
* **Development**: **Cursor / Claude code**
    * AIによる高速な「疑似言語インタープリタ（実行エンジン）」の実装。
